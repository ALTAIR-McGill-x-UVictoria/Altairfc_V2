MAX-M10M GPS — System Setup (Ubuntu, Raspberry Pi 4B)
=====================================================

Hardware
--------
- GPS module I2C address : 0x42  (u-blox DDC interface on /dev/i2c-1)
- Time pulse output      : GPIO18 / PCM_CLK  -> /dev/pps0 via pps-gpio overlay

1. Enable pps-gpio kernel overlay
----------------------------------
The dtoverlay line is already configured in /boot/firmware/config.txt:

    dtoverlay=dwc2,dr_mode=peripheral,pps-gpio,gpiopin=18

The overlay loads correctly and creates a pps@12 device tree node
(0x12 = GPIO18). However, the pps_gpio kernel module is not loaded
automatically — it must be explicitly loaded.

Load it now:
    sudo modprobe pps_gpio
    ls /dev/pps0              # should now exist

Make it load on every boot:
    echo pps_gpio | sudo tee -a /etc/modules

Verify PPS pulses (requires GPS to have a fix):
    sudo apt install pps-tools
    sudo ppstest /dev/pps0    # should show 1 Hz pulses

2. Install packages
--------------------
    sudo apt install gpsd gpsd-clients chrony pps-tools

3. Configure chrony for PPS discipline
----------------------------------------
gpsd is NOT required. The PPS signal on GPIO18 goes directly to /dev/pps0
and chrony reads it independently. NTP provides the coarse time reference
(which second the PPS pulse belongs to); PPS provides sub-microsecond
precision on top of that.

Add to /etc/chrony/chrony.conf:

    pool ntp.ubuntu.com iburst
    refclock PPS /dev/pps0 refid PPS precision 1e-7

Do not use "lock GPS" — that requires an SHM/gpsd source which is not
configured here. Chrony uses NTP for coarse lock and PPS for precision.

Restart chrony:
    sudo systemctl restart chrony
    sudo systemctl enable chrony

Verify:
    chronyc sources -v
    # PPS source should show * (selected) with very low offset once locked
    # This requires: GPS has a fix, pps_gpio module loaded, NTP reachable

5. Flight computer GPS driver
-------------------------------
The C driver (altairfc/drivers/gps_driver.c) talks directly to /dev/i2c-1
independently of gpsd. Both can coexist: the I2C DDC interface allows
multiple readers (each poll is stateless).

Build on the Pi (run once after deployment):
    cd altairfc/drivers
    bash build_gps.sh

Smoke test:
    cd altairfc
    python -c "from drivers.gps_driver import GpsDriver; d=GpsDriver(); print(d.read()); d.close()"

The GpsTask writes to DataStore keys "gps.*" at 1 Hz.
System clock sync (via chrony/PPS) is independent of the task loop.
